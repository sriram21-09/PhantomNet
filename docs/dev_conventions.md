\# Development Conventions



\## Branch Naming

feature/<name>

fix/<name>

docs/<name>



\## File Naming

snake\_case for Python files

kebab-case for frontend files



\## Commit Message Format

\[type] short description



Types:

\- feat: new feature

\- fix: bug fix

\- docs: documentation

\- chore: cleanup

\- refactor: code restructure



Example:

feat: add ssh honeypot tcp listener

docs: update architecture diagram



\## PR Rules

\- Every PR must have at least 1 reviewer.

\- PR description must include:

&nbsp; - What was changed

&nbsp; - Why it was changed

&nbsp; - How to test it
