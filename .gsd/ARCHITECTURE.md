## Overview

PhantomNet is an AI-driven active cyber deception and defense platform. It combines a network intrusion detection sniffer, real-time machine learning threat analysis, and automated response capabilities with interactive decoy honeypots. The system displays network activity, SDN topology state, threat metrics, and honeypot events via a modern web-based dashboard.

## Components

### Frontend Dashboard
- **Purpose:** Renders the system health, real-time threat map, incident feed, honeypot activity, and system topology.
- **Location:** `frontend-dev/phantomnet-dashboard`
- **Dependencies:** `react`, `react-router-dom`, `recharts`, `@xyflow/react`, `lucide-react`, `tailwindcss`, `leaflet`, `axios`

### FastAPI Backend
- **Purpose:** Serves REST API endpoints, coordinates WebSocket live metrics and event streaming, runs the real-time background sniffer, and triggers active defense actions.
- **Location:** `backend/`
- **Dependencies:** `fastapi`, `uvicorn`, `psutil`, `scapy`, `sqlalchemy`, `paramiko`, `cryptography`

### ML Threat Scoring Engine
- **Purpose:** Extracted features from packet streams are analyzed via pre-trained models (Random Forest, Isolation Forest, LSTM) to output threat scores and attack classifications (e.g. DDoS, SYN Scan).
- **Location:** `ml/` and `backend/ml_engine/`
- **Dependencies:** `scikit-learn`, `numpy`, `pandas`, `shap`, `joblib`, `skops`

### Honeypot Services
- **Purpose:** Simulation of vulnerable interfaces (SSH on port 2222, HTTP on port 8080, FTP on port 2121, SMTP on port 2525) to decoy attackers, capturing command execution history and credentials.
- **Location:** `honeypots/`
- **Dependencies:** `asyncssh`, `pyftpdlib`

### Network Simulation & Topology
- **Purpose:** Simulates enterprise network fabrics with multi-node mesh topologies and software-defined network controllers (POX/Mininet).
- **Location:** `topology/` and `simulation/`
- **Dependencies:** `mininet`, `dpkt`

### Automation
- **Purpose:** Scripting tools for managing sprint issues, milestones, and project board synchronization.
- **Location:** `automation/sprint/`
- **Dependencies:** Python CLI, `gh` CLI

## Data Flow

1. **Capture:** The `RealTimeSniffer` captures raw packets from the network interface using Scapy.
2. **Analysis:** Features are extracted and sent to the ML Threat Analyzer service, which runs model inference to score and classify malicious traffic.
3. **Storage & Audit:** Enriched logs are written to the PostgreSQL database, and anomalous events are logged to the security audit trail.
4. **Real-Time Push:** The backend pushes metrics updates (system health, honeypot connections, threat scores) via WebSockets to the React frontend.
5. **Defense Execution:** If a severe threat is detected or manually requested, the backend triggers firewall rules (IP blocking) and logs it under automated response history.
